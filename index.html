Array State Operations

ADD object 
setItems(prev => [
  ...prev,
  newItem
]);
DELETE by id 
setItems(prev =>
  prev.filter(item =>
    item.id !== id
  )
);
TOGGLE status 
setItems(prev =>
  prev.map(item =>
    item.id === id
      ? { ...item,
          active: !item.active }
      : item
  )
);
💡 Rule of thumb: Always use prev => callback form. Never mutate state directly. Spread ...item on toggle — never mutate the object itself.

useEffect Patterns

Async fetch inside useEffect (correct pattern) 
useEffect(() => {
  let cancelled = false;

  const fetchData = async () => {
    const response = await fetch("https://api.example.com/user");
    const data = await response.json();  // double await!
    if (!cancelled) setUser(data);       // guard unmounted
  };

  fetchData();  // don't forget to invoke!

  return () => { cancelled = true; };  // cleanup here, not inside async fn
}, []);
❌ Wrong — async directly on useEffect 
// React expects: undefined | () => void
// async fn returns a Promise — React ignores it, cleanup is lost
useEffect(async () => {        // ← NEVER do this
  const data = await fetchSomething();
  setData(data);
}, []);

Destructuring & Spread Tricks

💡 All examples below use this stub:

const user = {
  id: 1, name: 'Phani', email: '[email protected]',
  password: 'supersecret123', role: 'admin',
  address: { city: 'Dallas', zip: '75001' }
};

const items = [10, 20, 30, 40];
Remove key (omit password) 
const { password, ...safeUser } = user;
// safeUser = { id, name, email, role, address }
// password gone — never send to client

// inline in function param:
const send = ({ password, ...safeUser }) => safeUser;
Rename while destructuring 
const {
  name: userName,
  id: userId
} = user;
// userName = 'Phani'
// userId  = 1
Default values 
const { name, role = 'viewer' } = user;
// if user.role is undefined → 'viewer'
// user has role='admin' so → 'admin'
Rename + default combined 
const {
  name: userName = 'Anonymous'
} = user;
// userName = 'Phani' (default unused)
// if name missing → 'Anonymous'
Nested destructuring 
const {
  address: { city, zip }
} = user;
// city = 'Dallas', zip = '75001'
// address itself not in scope
Merge objects (last wins) 
const defaults = { role: 'viewer', active: true };
const merged = { ...defaults, ...user };
// user.role = 'admin' overrides defaults.role
Array — skip index 
const [first, , third] = items;
// first = 10, third = 30
// index 1 (20) skipped with empty comma
Swap without temp var 
let a = 1, b = 2;
[a, b] = [b, a];
// a = 2, b = 1 ✅
// no temp variable needed
Destructure function params (React staple) 
// instead of props.name, props.role inside body
const UserCard = ({ name, role = 'dev', address: { city } }) => (
  <div>{name} · {role} · {city}</div>
);

// usage:
<UserCard name="Phani" address={{ city: "Dallas" }} />
